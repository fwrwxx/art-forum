🔗 API Endpoints

Аутентифікація

POST /api/auth/register - Реєстрація
POST /api/auth/login - Вхід
POST /api/auth/logout - Вихід
POST /api/auth/refresh-token - Оновлення токену
GET /api/auth/verify-email/:token - Підтвердження email
POST /api/auth/forgot-password - Запит на відновлення
POST /api/auth/reset-password/:token - Скидання пароля

Профіль

GET /api/profile/me - Отримання профілю
PUT /api/profile/me - Оновлення профілю
PUT /api/profile/me/change-password - Зміна пароля

Управління користувачами (Admin)

GET /api/users - Всі користувачі
GET /api/users/:id - Користувач за ID
PUT /api/users/:id - Оновлення користувача
DELETE /api/users/:id - Видалення користувача

❓ Контрольні питання

  1. Що таке REST API та які його основні принципи? - REST API - це архітектурний стиль взаємодії клієнта і сервера через HTTP. Основні принципи: клієнт-серверна архітектура, stateless (без стану), використання HTTP методів, уніфікований інтерфейс.
  2. Які HTTP-методи використовуються в REST API? - GET (отримання), POST (створення), PUT/PATCH (оновлення), DELETE (видалення).
  3. Що таке реєстрація та авторизація? - Реєстрація - створення нового користувача. Авторизація - перевірка облікових даних та надання доступу.
  4. Для чого використовується JWT? - JWT використовується для створення безпечних токенів аутентифікації, які містять інформацію про користувача.
  5. Що таке middleware в Express? - Middleware - це функції, які виконуються між отриманням запиту та відправкою відповіді.
