🔗 API Endpoints

POST /api/auth/register - Реєстрація користувача
POST /api/auth/login - Вхід в систему
GET /api/books - Отримання списку книг (з пагінацією)
GET /api/books/:id - Отримання книги за ID
POST /api/books - Додавання книги (Admin)
PUT /api/books/:id - Оновлення книги (Admin)
DELETE /api/books/:id - Видалення книги (Admin)
GET /api-docs - Swagger документація
GET /health - Health check

🛡️ Перевірка захисту API

✅ Реалізовані заходи безпеки:

  • Helmet.js - захист HTTP-заголовків (перевірено через curl -I)
  • Rate Limiting - обмеження 100 запитів за 15 хвилин
  • JWT автентифікація - токени верифікуються (перевірено при логіні)
  • bcryptjs - паролі хешуються (12 солей)
  • CORS - налаштовано крос-доменний доступ
  • Валідація - перевірка email, password, полів книг

❓ Контрольні запитання

Що таке Helmet у Node.js?
Модуль для захисту HTTP-заголовків, запобігає XSS, clickjacking та іншим атакам.
Для чого використовується rate limiting?
Обмеження кількості запитів з однієї IP-адреси для захисту від DDoS та brute-force атак.
Які переваги кешування?
Зменшення навантаження на сервер, пришвидшення відповіді, покращення масштабованості.
Чим відрізняється unit testing від integration testing?
Unit тестує окремі функції, Integration перевіряє взаємодію компонентів.
Для чого потрібна валідація даних?
Запобігання ін'єкціям, захист від некоректних даних, забезпечення цілісності БД.
Що таке JWT?
JSON Web Token - безпечний спосіб передачі даних між клієнтом та сервером для автентифікації.
Які типи кешування існують?
In-memory, Redis, CDN, Browser, Database query cache.
Які існують способи оптимізації API?
Пагінація, індексація, кешування, стиснення, асинхронна обробка, оптимізація запитів.
